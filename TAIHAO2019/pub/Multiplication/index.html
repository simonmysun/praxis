There is a powerful tool called Regular expressions(regex). A good regex might save 90+% of processing time and 90+% lines of code. Try to solve this problem with regex.

Problem description

Match correct multiplication statements in the domain ^x+\*x+=x+$ using only the ECMAScript subset of regex functionality.

Testing

Two columns of example strings are shown in order to visualize the correctness of your regex. You should not only match or not match them. The final test includes more test cases.

You should match:

  • x*x=x
  • xx*x=xx
  • x*xxx=xxx
  • xxxx*xxx=xxxxxxxxxxxx
  • xxx*xxxx=xxxxxxxxxxxx
  • xxxxxxx*x=xxxxxxx
  • xxxxxx*xxx=xxxxxxxxxxxxxxxxxx
  • xxxx*xxxx=xxxxxxxxxxxxxxxx
  • xx*xxxxx=xxxxxxxxxx

You should not match:

  • x*x=xx
  • xx*x=xxx
  • x*xxx=xx
  • xxxx*xxx=xxxxxxxxxxxxx
  • xxx*xxxx=xxxxxxxxxxx
  • xxxxxxx*x=x
  • xxxxxx*xxx=xxxxxxxxxxxx
  • xxxx*xxxx=xxxxxxxxxxxxxxx
  • xx*xxxxx=xxxxxxxxxxx

Once your submission satisfies the conditions, you will get the key for this level.
